Optimum ERP

Streamlining Business Processes with the MERN Stack

About

Optimum ERP is a full-stack ERP platform for managing business operations such as users, organizations, invoicing, purchases, expenses, reports, and settings. This project is open source.

Features

  • Expense Management
  • Product Management
  • Quotation Handling
  • Purchase and Sales Operations
  • GST Management
  • Expense Categorization
  • User Authentication
  • PDF Generation
  • Role-Based Access Control
  • Notification System
  • Audit Trails
  • Multi-Tenancy

Technologies Used

Optimum ERP is built with:

  • Frontend: React + Vite
  • Backend: Node.js + Express
  • Database: MongoDB
  • Scheduler: Background cron service
  • Deployment: Docker Compose
  • Chakra UI
  • Framer Motion
  • PDF Generation using PDF Shift

Deployment Steps

  1. Prerequisites
    • Install Docker
    • Install Docker Compose plugin
  2. Clone the repository: 
    git clone https://github.com/Atuldubey98/OptimumERP.git
  3. Open the project directory: 
    cd OptimumERP
  4. Create a .env file in the project root using env.txt as reference. Add the following keys: 
    MONGO_URI=mongodb://mongo:27017/mernapp?replicaSet=rs0
SESSION_SECRET=your-secure-random-string
VITE_API_URL=http://localhost:3000
IMPORT_CRON_SCHEDULE=*/5 * * * *
LOG_FILE_PATH=./logs/app.log
DEFAULT_USER_PLAN=platinum
NETWORK_STORAGE_PATH=./uploads
  5. Start all services: 
    docker compose up -d --build
  6. Verify services: 
    docker compose ps
  7. Verify backend health: 
    curl http://localhost:3000/api/v1/health
  8. Access the application:
    • Frontend: http://localhost:5173
    • Backend API: http://localhost:3000/api/v1

Common Commands

  • Restart services: 
    docker compose restart
  • Stop services: 
    docker compose down
  • View logs: 
    docker compose logs -f

Production Notes

  • Keep .env secrets private.
  • Use HTTPS with a reverse proxy for production.
  • Back up MongoDB data volume regularly.
  • Rotate credentials periodically.

API Documentation

The backend exposes a REST API under the base path /api/v1. Authentication uses session-based cookies (express-session).

Authentication

  1. Call the login endpoint to create a session.
  2. Store the session cookie returned by the server.
  3. Send that cookie on all protected API calls.
# Login and store cookies
curl -i -c cookies.txt \
  -X POST http://localhost:3000/api/v1/users/login \
  -H "Content-Type: application/json" \
  -d '{"email":"you@example.com","password":"your-password"}'

# Use stored cookies for protected endpoints
curl -i -b cookies.txt \
  http://localhost:3000/api/v1/users

Key Endpoints

  • GET /api/v1/health — Service health check
  • POST /api/v1/users/register — Register a new user
  • POST /api/v1/users/login — Login and create session
  • GET /api/v1/organizations — List organizations for current user
  • GET /api/v1/organizations/:orgId/invoices — List invoices
  • GET /api/v1/organizations/:orgId/purchases — List purchases
  • GET /api/v1/organizations/:orgId/expenses — List expenses
  • GET /api/v1/organizations/:orgId/dashboard — Dashboard summary
  • GET /api/v1/organizations/:orgId/reports/:reportType — Reports

For the full API reference, see backend/README.md.
